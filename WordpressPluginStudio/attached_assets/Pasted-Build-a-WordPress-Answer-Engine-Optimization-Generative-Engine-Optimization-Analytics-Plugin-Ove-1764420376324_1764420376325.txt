```
### Backend Integration
The plugin connects to an existing Express.js backend with these endpoints:

**Base URL:** Configurable (e.g., `https://pleasant-optimism-production.up.railway.app`)

**API Endpoints:**
1. `POST /api/analyze` - Main content analysis
   - Actions: `full_analysis`, `suggest_titles`, `generate_description`, `summarize_content`, `rewrite_sentence`
   - Headers: `Authorization: Bearer {api_key}`
   - Body: `{ action, content, industry, sentence }`

2. `GET /api/users/me` - User verification and usage info
   - Headers: `Authorization: Bearer {api_key}`
   - Returns: `{ id, email, subscriptionStatus, usage: { count, limit } }`

**Authentication:**
- Bearer token authentication
- API keys format: `rain_os_key_[48_hex_characters]`

**Error Handling:**
- 401: Invalid API key
- 402: Payment required (inactive subscription)
- 429: Rate limit exceeded
- 500: Server error
```

```
## Data Flow Example
```
1. User writes post in WordPress editor
2. User scrolls to "Rain OS SEO Analysis" meta box
3. User selects industry (dropdown)
4. User clicks "Analyze Content" button
5. JavaScript captures content from editor
6. AJAX request to WordPress backend
7. PHP makes HTTP POST to backend API
8. Backend analyzes with Google Gemini
9. Response returns to PHP
10. PHP returns to JavaScript
11. JavaScript renders beautiful results
12. Usage counter updates
13. Results saved to localStorage
