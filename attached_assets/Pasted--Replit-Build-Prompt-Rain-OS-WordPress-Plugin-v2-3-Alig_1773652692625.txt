# Replit Build Prompt: Rain OS WordPress Plugin v2.3 Alignment

## Context

The Rain OS backend has been upgraded to v2.3 with:
- **4 pillars** instead of 3 (added: Product Discoverability)
- **14 new Phase 2 sub-scores** (semantic, authority, discoverability metrics)
- **New `/api/url-scan` endpoint** for scoring URLs directly
- **GET `/api/capabilities`** endpoint to discover API features

The WordPress plugin is still hardcoded for **3 pillars** and has **no URL scanning support**. This breaks compatibility and hides the new 4th pillar from users.

## Implementation Notes

### 1. Backward Compatibility
- ✅ Plugin MUST work with both v2.1 (3 pillar) and v2.3 (4 pillar) APIs during transition
- If response includes `productDiscoverability`, display 4 pillars; otherwise display 3
- Handle missing `phase2_sub_scores` gracefully (don't throw errors)

### 2. Data Migration
- Old analysis history will have `product_discoverability = NULL` → display as 0 or "N/A"
- New analyses will populate all 4 pillars
- No need to backfill old data

### 3. Testing Checklist
- [ ] Mock response with all 4 pillars + phase2_sub_scores → UI displays correctly
- [ ] Mock response with 3 pillars only (v2.1 compat) → UI displays 3 pillars
- [ ] URL scan response → technical signals display correctly
- [ ] Dashboard averages → correctly sum/avg all 4 pillars
- [ ] Historical data → renders without errors even if `product_discoverability` is NULL
- [ ] Multiple tabs (Gutenberg sidebar, Admin dashboard) → all show consistent data

### 4. Constants to Define
```php

define( 'RAIRO_PILLARS_COUNT', 4 );     // Dynamic, but hardcoded fallback
define( 'RAIRO_CAPABILITY_CACHE_TTL', 6 * HOUR_IN_SECONDS );
```
