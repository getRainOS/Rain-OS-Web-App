#6 - Full Plugin Polish & Prototype Sync
What & Why
The live plugin looks flat and unsophisticated compared to the prototype (App.jsx). Cards are plain solid-color rectangles with no depth. Text using rgba(255,255,255,0.5) is illegible. Toggle switches show both a raw HTML checkbox and an unstyled circle simultaneously. The Pillar Breakdown score display is a filled blob. The Plugin URI links to the wrong domain. The prototype itself also needs to reflect the same polished look.

All 12 plugin templates share the same CSS (admin.css) and card/component classes, so upgrading the shared CSS components cascades polish to every page. Targeted structural changes are needed only for the Pillar Breakdown score circle. The URL Scanner has its own url-scanner.css that also needs a pass.

Done looks like
Plugin — all pages:

Every .rain-os-card has a glassmorphism treatment: translucent gradient background (rgba(255,255,255,0.03) base with subtle color tint), backdrop-filter: blur(8px), a soft box-shadow, and rounded corners — consistent across Dashboard, Settings, Content Analyzer, Score History, Documentation, Help, Upgrade, Learn AI Readability, Content Signals, Technical Signals
All muted/secondary body text is clearly legible (minimum rgba(255,255,255,0.75) on dark backgrounds)
KPI cards on the Dashboard have a hover lift effect and a subtle colored top-border accent in their pillar color
The Score History table rows have alternating subtle background tints and hover states
Content Signals and Technical Signals sections use the same glassmorphism card treatment
Plugin — specific pages:

Settings: toggle switches are smooth pill sliders (square checkbox hidden, styled thumb slides, checked = orange glow)
Pillar Breakdown: overall score is a ring/arc (conic-gradient or SVG stroke) not a filled circle; pillar cards have per-color gradient glass backgrounds; pillar score numbers render in their pillar color; progress bars are thinner (6px) with gradient fills
URL Scanner: cards in url-scanner.css updated to match the glassmorphism treatment of the main CSS
Plugin — metadata:

"Visit plugin site" link in the WordPress plugins list opens https://www.getrainos.com (Plugin URI and Author URI both corrected)

All pages (Dashboard, Performance, Pillar Breakdown, Category Scores, Content Signals, Learn AI Readability, Improve Score, URL Scanner, Settings, Docs, Upgrade, Gutenberg Sidebar) use the same glassmorphism card style, refined text contrast, and polished component look as the updated plugin
The overall visual language matches: same colors, same card depth, same typography weight
