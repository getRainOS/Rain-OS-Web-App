3) The safest pattern (so nothing breaks)
DO NOT put the fetch directly into UI components.
Instead:
Add a new function in useAIReadiness.js
Something like:
•	refreshBackendAnalysis(postId)
•	fetches: wp.apiFetch({ path: '/rain-os-aeo/v1/backend-analysis/' + postId })
•	if null/204 -> do nothing
•	if data exists -> merge into state
Then useContentAnalysis.js simply calls it after completion.
This keeps responsibilities clean and additive.
